# Gaps Formatting Rules

Rules for `gaps.md` artifact.

## Structure

```markdown
# Gaps

| # | Type | Artifact | Description | Parent | Suggested tree question |
|---|------|----------|-------------|--------|-------------------------|
| 1 | GAP | invariants | No invariants for Strategy | AD-007 | ? What must always be true for Strategy state? |
| 2 | CHOICE | components | Oracle logic placed in Strategy | AD-012 | ? Where does oracle integration live? |
```

## Rules

- Type: `GAP` (information missing) or `CHOICE` (ambiguity, summarizer picked one interpretation).
- Parent: `AD-NNN` of the nearest existing node where the question belongs. `—` if no natural parent.
- Generated by the `gen-gaps` pipeline step, which walks every prior artifact and every spec. Empty file (with a "No gaps" line) if nothing is marked.
