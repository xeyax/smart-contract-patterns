# Overview Formatting Rules

Rules for `overview.md` artifact.

## Structure

```markdown
# Overview

2-3 sentences: what the system does, for whom, on which chain.

## Key Decisions

- **Decision name**: one-line rationale.
- **Decision name**: one-line rationale.
```

## Rules

- Key decisions = architectural choices with trade-offs, not implementation details.
- Each decision = one bullet, bold name + rationale after colon.
- These are candidates for formal ADRs — keep them crisp.
