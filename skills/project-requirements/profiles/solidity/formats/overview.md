# overview.md formatting rules

## Structure

```markdown
# Overview

## What & for whom
2–3 sentences. The system does X for audience Y.

## Context
Standalone system, OR part of larger system Z, where it plays role R.

## External dependencies
- **Dep 1** — one-line role
- **Dep 2** — one-line role
```

## Rules

- Pull from Purpose verbatim where possible.
- Keep total length ≤ 1 page. Use bullets, not paragraphs, after the first block.
- If Purpose has no context section → `[GAP]: context not stated in Purpose (standalone vs part-of-larger)`.
