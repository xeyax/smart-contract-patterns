# Artifact Formatting Rules

Exact formatting conventions for generated artifacts. The summarizer follows these when generating, the reviewer checks compliance.

Individual format files:

- [mermaid.md](artifact-formats/mermaid.md) — all mermaid diagrams (no semicolons, one POST per Note, participant consistency)
- [overview.md](artifact-formats/overview.md) — overview structure, key decisions format
- [contracts.md](artifact-formats/contracts.md) — state variable annotation style
- [interfaces.md](artifact-formats/interfaces.md) — access modifier placement, NatSpec, section headers
- [call-diagrams.md](artifact-formats/call-diagrams.md) — postconditions, quantitative POSTs, operation coverage
- [token-flows.md](artifact-formats/token-flows.md) — diagram style, text summaries
- [access-control.md](artifact-formats/access-control.md) — table format, guard column style
- [state-machines.md](artifact-formats/state-machines.md) — state diagram, transition table, invariants
- [invariants.md](artifact-formats/invariants.md) — numbering, format
- [risks.md](artifact-formats/risks.md) — risk table, source/mitigation/status
- [plan.md](artifact-formats/plan.md) — task table, dependency graph
- [specs.md](artifact-formats/specs.md) — section titles, traceability comments, naming, mathematical properties
- [gaps.md](artifact-formats/gaps.md) — gap table, parent field
