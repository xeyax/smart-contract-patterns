# Vision -- Reviewer

```
Review the Vision document. Work ONLY with the document below, do not trust the creator's claims.
Your task is to CUT excess, not add. Every element must justify its existence.

Document:
{{VISION_DOC}}

## Critical Thinking

For EACH element of the document, run 5 tests:
1. "So What?" -- why is this needed? if unclear -- remove
2. "What If We Don't?" -- what happens without this? if nothing -- remove
3. "Simpler?" -- can it be formulated more simply?
4. "Who Benefits?" -- which specific persona benefits? if none -- remove
5. "Can We Measure?" -- can it be verified in practice?

## Checklist (PASS/FAIL + quote)

1. Is the problem specific? (not generic words -- quote it)
2. Users with context? (not just a role -- quote it)
3. No implementation details? (technologies, protocols, languages)
4. Do non-goals cut off directions?
5. No contradictions? (goals vs non-goals, problem vs metrics -- list the checked pairs)
6. No duplication? (one idea in two places)
7. Is the document concise? (no fluff, no unnecessary explanations, every sentence carries meaning)

## Metrics -- 4 questions for EACH

| Metric | Who Cares? (persona) | Measurable? (how to measure) | Not Vanity? (outcome, not activity) | Comparable? (baseline) | PASS/FAIL |
|--------|---------------------|------------------------------|-------------------------------------|----------------------|-----------|

Vanity traps: TVL, transaction count, user count -- these are activity, not outcome. Remove or reformulate.
There should be 2-4 metrics. More than that -- suggest what to remove.

## Demand analysis

Briefly: under what conditions is the product in demand? Under what conditions -- NOT?

## Conclusion

Find at least 3 problems or explain why there are none. For each FAIL -- a recommendation.

Verdict: ACCEPTED / NEEDS REVISION.
```
