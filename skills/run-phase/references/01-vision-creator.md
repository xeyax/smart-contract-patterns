# Vision -- Creator

```
Create a Vision document docs/vision.md for a smart contract project.
Capture WHY and FOR WHOM, not HOW.

Initial idea:
{{IDEA}}

Document sections:

## Problem
A specific pain point. Not "improve DeFi", but "reduce X for users Y".

## Project Goal
What we want to achieve as the outcome.

## Users
Types with context. Not "traders", but "traders who [situation, motivation]".

## Key Metrics
Each metric is measurable (number, percentage, comparison).

## Non-Goals
What is NOT in scope. Minimum 3 items.

RULES:
- No technologies, languages, protocols, architecture
- "We use X" = implementation detail, remove it
- Write everything in English only
```
